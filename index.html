React?

"A JavaScript library for building user interfaces."

Made by Facebook

Just the UI

Makes no assumptions about the rest of the application.

It's the V in MVC.

Super simple

No overload of features to explore (and get lost in).

Fast to start using.

Components

React is all about building components.

Components are not templates.

JSX

Components unify markup with its view logic.

(Yes, JavaScript and HTML in the same file. Yuck...)

Single responsibility

Components should be encapsulated and reusable.

Makes them easier to maintain.

Properties and state

A component's properties never change.

State is mutable and its changes affect the view.

It's all about state

When some data changes, the view will update to represent the new state.

These are called "Reactive updates".

Release the data!

There's no need for data binding, since data changes affect the view directly.
